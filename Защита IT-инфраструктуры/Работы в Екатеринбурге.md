# Настройка подчиненного ЦС на YEKT-RTR

## Настройка ЦС

Проверяем, чтобы был ```openssl```, если нет, ставим:

```
apt install openssl -y
```

Создаем структуру каталога для ЦС:

```
mkdir -p /sub/ca/{certs,crl,newcerts,private}
```

```
chmod 700 /sub/ca/private/
```

```
touch /sub/ca/index.txt
```

```
echo 1000 > /sub/ca/serial
```

```
echo 1000 > /sub/ca/crlnumber
```

Создаем конфиг ЦС:

```
nano /sub/ca/openssl.cnf
```

```
[ ca ]
default_ca = CA_default

[ CA_default ]
dir               = /sub/ca
certs             = $dir/certs
crl_dir           = $dir/crl
crlnumber         = $dir/crlnumber
database          = $dir/index.txt
new_certs_dir     = $dir/newcerts
serial            = $dir/serial

certificate       = $dir/certs/subca.crt
private_key       = $dir/private/subca.key

default_md        = sha256
default_days      = 825
copy_extensions   = copy
unique_subject    = no
policy            = policy_loose


[ policy_loose ]
commonName              = supplied

[ req ]
default_bits        = 4096
default_md          = sha256
distinguished_name  = req_distinguished_name

[ req_distinguished_name ]
commonName          = Common Name

[ v3_intermediate_ca ]
subjectKeyIdentifier   = hash
authorityKeyIdentifier = keyid:always,issuer
basicConstraints       = critical, CA:true, pathlen:0
keyUsage               = critical, digitalSignature, cRLSign, keyCertSign
```

Генерируем закрытый ключ ЦС:

```
openssl genrsa -out /sub/ca/private/subca.key 4096
```

Создаем сертификат ЦС:

```
openssl req -config /sub/ca/openssl.cnf -new -sha256 -key /sub/ca/private/subca.key -out subca.csr -subj "/C=RU/ST=Moskovskaya/L=Moskva/O=Cool CA/CN=Yekaterinburg CA"
```

Переносим полученный ```subca.csr``` на ```MSK-DC1```:

```
scp subca.csr user@192.168.1.2:/home/user/subca.csr
```

На ```MSK-DC1``` одписываем основным ЦС:

```
openssl ca -config /root/ca/openssl.cnf -extensions v3_intermediate_ca -days 1825 -notest -md sha256 -in subca.csr -out subca.crt
```

Возвращаем подписанный сертификат ```subca.crt``` на ```YEKT-RTR```:

```
scp subca.crt user@192.168.2.1:/home/user/subca.crt
```

Кладем подписанный сертификат в ```/sub/ca/certs```:

```
mv subca.crt /sub/ca/cert/subca.crt
```

## Выпуск сертификатов для web-сервисов

### billing.effort.cool (сайт на YEKT-BILLING)

Генерим ключ:

```
openssl genrsa -out billing.effort.cool.key 4096
```

Создаем конфиг:

```
nano billing.effort.cool.cnf
```

```
[ req ]
distinguished_name = req_distinguished_name
req_extensions = server_cert

[ req_distinguished_name ]

[ server_cert ]
basicConstraints = CA:FALSE
keyUsage = digitalSignature, keyEncipherment
extendedKeyUsage = serverAuth
subjectAltName = @alt_names

[ alt_names ]
DNS.1 = billing.effort.cool
IP.1 = 192.168.2.100
```

Делаем запрос сертификата:

```
openssl req -new -key billing.effort.cool.key -out billing.effort.cool.csr -config billing.effort.cool.cnf -subj "/C=RU/ST=Moskovskaya/L=Moskva/O=Cool CA/CN=billing.effort.cool"
```

Подписываем сертификат:

```
openssl ca -config /sub/ca/openssl.cnf -extfile billing.effort.cool.cnf -extensions server_cert -notext -in billing.effort.cool.csr -out billing.effort.cool.crt
```

Собираем chain:

```
cat billing.effort.cool.crt /sub/ca/certs/subca.crt > billing.effort.cool.fullchain.crt
```

Затем переносим ```billing.effort.cool.key``` и ```billing.effort.cool.fullchain.crt``` на YEKT-BILLING и используем для настройки HTTPS.
