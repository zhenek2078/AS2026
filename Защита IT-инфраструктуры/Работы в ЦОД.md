# Настройка PAT через NFTABLES

На каждом роутере включаем IP-forward:

```
echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf
```

И перезапускаем:

```
sysctl -p
```

## DC-RTR-1 и DC-RTR-2

Включаем nftables:

```
systemctl enable nftables && systemctl restart nftables
```

Создаем таблицу nat:

```
nft add table ip nat
```

Добаялвем цепочки:

```
nft add chain ip nat prerouting { type nat hook prerouting priority 0 \; }
```

```
nft add chain ip nat postrouting { type nat hook postrouting priority 100 \; }
```

Добавляем правило:

```
nft add rule ip nat postrouting oifname "ens3" ip saddr 10.15.10.0/24 masquerade
```

Сохраняем правила:

```
nft list ruleset > /etc/nftables.conf
```

Перезапускаем nftables:

```
systemctl restart nftables
```

# Настройка wireguard между DC-RTR-1, DC-RTR-2, MSK-RTR и YEKT-RTR

Ставим на все роутеры wireguard:

```
apt install wireguard -y
```

Генерируем ключи:

## На DC-RTR-1:

Для DC-RTR-1 <-> MSK:

```
wg genkey | tee msk_priv_dc1 | wg pubkey > msk_pub_dc1
```

Для DC-RTR-1 <-> YEKT:

```
wg genkey | tee yekt_priv_dc1 | wg pubkey > yekt_pub_dc1
```

## На DC-RTR-2:

Для DC-RTR-2 <-> MSK:

```
wg genkey | tee msk_priv_dc2 | wg pubkey > msk_pub_dc2
```

Для DC-RTR-2 <-> YEKT:

```
wg genkey | tee yekt_priv_dc2 | wg pubkey > yekt_pub_dc2
```

## На MSK-RTR:

Для MSK <-> DC-RTR-1:

```
wg genkey | tee dc1_priv_msk | wg pubkey > dc1_pub_msk
```

Для MSK <-> DC-RTR-2:

```
wg genkey | tee dc2_priv_msk | wg pubkey > dc2_pub_msk
```

## На YEKT-RTR:

Для YEKT <-> DC-RTR-1:

```
wg genkey | tee dc1_priv_yekt | wg pubkey > dc1_pub_yekt
```

Для YEKT <-> DC-RTR-2:

```
wg genkey | tee dc2_priv_yekt | wg pubkey > dc2_pub_yekt
```

Далее настраиваем туннельные интерфейсы:

## На DC-RTR-1:

Для DC-RTR-1 <-> MSK:

```
nano /etc/wireguard/wg-msk.conf
```

```
[Interface]
Address = 10.7.7.1/30
PrivateKey = СОДЕРЖИМОЕ ИЗ msk_priv_dc1
ListenPort = 51820
Table = off

[Peer]
PublicKey = СОДЕРЖИМОЕ ИЗ dc1_pub_msk
Endpoint = 188.121.90.2:51820
AllowedIPs = 10.7.7.2/32, 224.0.0.0/24, 192.168.1.0/24
PersistentKeepalive = 25
```

Для DC-RTR-1 <-> YEKT:

```
nano /etc/wireguard/wg-yekt.conf
```

```
[Interface]
Address = 10.6.6.1/30
PrivateKey = СОДЕРЖИМОЕ ИЗ yekt_priv_dc1
ListenPort = 51821
Table = off

[Peer]
PublicKey = СОДЕРЖИМОЕ ИЗ dc1_pub_yekt
Endpoint = 88.8.8.27:51820
AllowedIPs = 10.6.6.2/32, 224.0.0.0/24, 192.168.2.0/24

Запускаем туннели:

```
wg-quick up wg-msk && systemctl start wg-quick@wg-msk
```

wg-quick up wg-yekt && systemctl start wg-quick@wg-yekt
```

# Настройка OSPF

На всех роутерах ставим пакет:

```
apt install frr -y
```

После установки редактируем конфиг:

```
nano /etc/frr/daemons
```

Заменяем ```ospfd=no``` на ```ospfd=yes```.

Перезапускаем:

```
systemctl restart frr
```

## DC-RTR-1

```
vtysh
  conf t
    router ospf
      passive-interface default
      network 10.6.6.0/30 area 0
      network 10.7.7.0/30 area 0
      network 10.15.10.0/24 area 0
    exit
    interface wg-msk
      multicast
      no ip ospf passive
      ip ospf authentication message-digest
      ip ospf message-digest-key 1 md5 C00lComp
      exit
    interface wg-yekt
      multicast
      no ip ospf passive
      ip ospf authentication message-digest
      ip ospf message-digest-key 1 md5 C00lComp
      exit
    do write
```

# Настройка VRRP между DC-RTR-1 и DC-RTR-2

Устанавливаем пакеты на оба роутера:

```
apt install keepalived -y
```

## DC-RTR-1

```
nano /etc/keepalived/keepalived.conf
```

```
vrrp_instance VI_0 {
	state MASTER
	interface ens4
	virtual_router_id 51
	priority 150
	advert_int 1
	virtual_ipaddress {
		10.15.10.1/24
	}
}
```

Перезапускаем сервис:

```
systelctm restart keepalived
```

## DC-RTR-2

```
nano /etc/keepalived/keepalived.conf
```

```
vrrp_instance VI_0 {
	state BACKUP
	interface ens4
	virtual_router_id 51
	priority 100
	advert_int 1
	virtual_ipaddress {
		10.15.10.1/24
	}
}
```

Перезапускаем сервис:

```
systelctm restart keepalived
```

# Настройка SSH

## На всех хостах (DC-RTR-1, DC-RTR-2, DC-STORAGE, DC-MAILSERVER:

Добавляем пользователя cod_admin:

```
adduser cod_admin
```

Пароль:

```
P@ssw0rd
```

## На DC-STORAGE и DC-MAILSERVER ставим пакет sudo:

```
apt install sudo -y
```

Открываем visudo:

```
visudo
```

Добавляем строку:

```
cod_admin ALL=(ALL:ALL) NOPASSWD: ALL
```

## Доступ SSH

### Генерация ключей:

На DC-STORAGE создаем ключи из-под cod_admin:

```
ssh-keygen
```

На все вопросы просто жмем ```Enter```.

Копируем ключи на DC-RTR-1, DC-RTR-2 и DC-MAILSERVER:

```
ssh_copy_id cod_admin@10.15.10.100
```

```
ssh_copy_id cod_admin@10.15.10.2
```

```
ssh_copy_id cod_admin@10.15.10.3
```

Переносим ключи в ```/ssh_keys```:

```
sudo mv .ssh/* /ssh_keys/
```

```
rm -r .ssh
```

### Доступ по локальным интерфейсам:

На DC-RTR-1:

```
nano /etc/ssh/sshd_config
```

```
ListenAddress = 10.15.10.3
```

Перезапускаем сервис:

```
systemctl restart sshd
```

На DC-RTR-2:

```
nano /etc/ssh/sshd_config
```

```
ListenAddress = 10.15.10.2
```

Перезапускаем сервис:

```
systemctl restart sshd
```

На DC-MAILSERVER:

```
nano /etc/ssh/sshd_config
```

```
ListenAddress = 10.15.10.100
```

Перезапускаем сервис:

```
systemctl restart sshd
```

### Харденинг SSH на DC-RTR-1, DC-RTR-2, DC-MAILSERVER:

Создаем файл:

```
nano /etc/ssh/sshd_config.d/atom.conf
```

Прописываем следующие настройки:

```
PasswordAuthentication no
PubkeyAuthentication yes
AuthenticationMethods publickey
PermitRootLogin no
MaxStartups 3
ClientAliveInterval 300
ClientAliveCountMax 0
```

Перезапускаем сервис:

```
systemctl restart sshd
```
