# Настройка PAT через NFTABLES

На каждом роутере включаем IP-forward:

```
echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf
```

И перезапускаем:

```
sysctl -p
```

## DC-RTR-1 и DC-RTR-2

Включаем nftables:

```
systemctl enable nftables && systemctl restart nftables
```

Создаем таблицу nat:

```
nft add table ip nat
```

Добаялвем цепочки:

```
nft add chain ip nat prerouting { type nat hook prerouting priority 0 \; }
```

```
nft add chain ip nat postrouting { type nat hook postrouting priority 100 \; }
```

Добавляем правило:

```
nft add rule ip nat postrouting oifname "ens3" ip saddr 10.15.10.0/24 masquerade
```

Сохраняем правила:

```
nft list ruleset > /etc/nftables.conf
```

Перезапускаем nftables:

```
systemctl restart nftables
```

# Настройка OSPF

На всех роутерах ставим пакет:

```
apt install frr -y
```

После установки редактируем конфиг:

```
nano /etc/frr/daemons
```

Заменяем ```ospfd=no``` на ```ospfd=yes```.

Перезапускаем:

```
systemctl restart frr
```

## DC-RTR-1

```
vtysh
  conf t
    router ospf
      passive-interface default
      network 10.6.6.0/30 area 0
      network 10.7.7.0/30 area 0
      network 10.15.10.0/24 area 0
    exit
    interface wg-msk
      multicast
      no ip ospf passive
      ip ospf authentication message-digest
      ip ospf message-digest-key 1 md5 C00lComp
      exit
    interface wg-yekt
      multicast
      no ip ospf passive
      ip ospf authentication message-digest
      ip ospf message-digest-key 1 md5 C00lComp
      exit
    do write
```

# Настройка VRRP между DC-RTR-1 и DC-RTR-2

Устанавливаем пакеты на оба роутера:

```
apt install keepalived -y
```

## DC-RTR-1

```
nano /etc/keepalived/keepalived.conf
```

```
vrrp_instance VI_0 {
	state MASTER
	interface ens4
	virtual_router_id 51
	priority 150
	advert_int 1
	virtual_ipaddress {
		10.15.10.1/24
	}
}
```

Перезапускаем сервис:

```
systelctm restart keepalived
```

## DC-RTR-2


```
vrrp_instance VI_0 {
	state BACKUP
	interface ens4
	virtual_router_id 51
	priority 100
	advert_int 1
	virtual_ipaddress {
		10.15.10.1/24
	}
}
```

Перезапускаем сервис:


# Настройка SSH

## Добавление пользователя cod_admin на всех хостах (DC-RTR-1, DC-RTR-2, DC-STORAGE, DC-MAILSERVER)

Добавляем пользователя cod_admin:

```
adduser cod_admin
```

Пароль:

```
P@ssw0rd
```

## sudo для cod_admin без пароля на DC-STORAGE и DC-MAILSERVER

Ставим пакет sudo:

```
apt install sudo -y
```

Открываем visudo:

```
visudo
```

Добавляем строку:

```
cod_admin ALL=(ALL:ALL) NOPASSWD: ALL
```

## Доступ SSH

### Генерация ключей:

На DC-STORAGE создаем ключи из-под cod_admin:

```
ssh-keygen
```

На все вопросы просто жмем ```Enter```.

Копируем ключи

- на DC-MAILSERVER:

```
ssh_copy_id cod_admin@10.15.10.100
```

- на DC-RTR-2:

```
ssh_copy_id cod_admin@10.15.10.2
```

- на DC-RTR-1:

```
ssh_copy_id cod_admin@10.15.10.3
```

Переносим ключи в ```/ssh_keys```:

```
sudo mv .ssh/* /ssh_keys/
```

```
rm -r .ssh
```

### Доступ по локальным интерфейсам:

**На DC-RTR-1:**

```
nano /etc/ssh/sshd_config
```

```
ListenAddress = 10.15.10.3
```

Перезапускаем сервис:

```
systemctl restart sshd
```

**На DC-RTR-2:**

```
nano /etc/ssh/sshd_config
```

```
ListenAddress = 10.15.10.2
```

Перезапускаем сервис:

```
systemctl restart sshd
```

**На DC-MAILSERVER:**

```
nano /etc/ssh/sshd_config
```

```
ListenAddress = 10.15.10.100
```

Перезапускаем сервис:

```
systemctl restart sshd
```

### Харденинг SSH на DC-RTR-1, DC-RTR-2, DC-MAILSERVER:

Создаем файл:

```
nano /etc/ssh/sshd_config.d/atom.conf
```

Прописываем следующие настройки:

```
PasswordAuthentication no
PubkeyAuthentication yes
AuthenticationMethods publickey
PermitRootLogin no
MaxStartups 3
ClientAliveInterval 300
ClientAliveCountMax 0
```

Перезапускаем сервис:

```
systemctl restart sshd
```

# Настройка Suricata

> Настройки оинаковые на DC-RTR-1 и на DC-RTR-2.

Ставим пакет:

```
apt install suricata -y
```

Включаем режим IDS и настраиваем интерфейсы:

```
nano /etc/suricata/suricata.yaml
```

```
af-packet:
  - interface: ens3
    cluster-id: 99
    cluster-type: cluster_flow
    defrag: yes

  - interface: wg-msk
    cluster-id: 98
    cluster-type: cluster_flow
    defrag: yes

  - interface: wg-yekt
    cluster-id: 97
    cluster-type: cluster_flow
    defrag: yes
```

> Также внизу этого блока надо удалить строку ```- interface: default```.

Подключаем правила:

```
nano /etc/suricata/suricata.yaml
```

Находим строку ```rule-files:``` и добавляем ```- local.rules```, удаляем ```- suricata.rules```.

Создаем правила:

```
nano /etc/suricata/rules/local.rules
```

```
# 1. Сканирование портов
alert tcp any any -> any any (msg:"PORT SCAN SYN"; flags:S; threshold:type both, track by_src, count 20, seconds 5; sid:1000001; rev:1;)

# 2. Атаки на VPN (подбор / флуд)
alert udp any any -> any 51820 (msg:"WireGuard brute force attempt"; threshold:type both, track by_src, count 15, seconds 10; sid:1000002; rev:1;)
alert udp any any -> any 51821 (msg:"WireGuard brute force attempt"; threshold:type both, track by_src, count 15, seconds 10; sid:1000003; rev:1;)

# 3. Нестандартные пакеты VPN
alert udp any any -> any 51820 (msg:"WireGuard malformed packet"; dsize:<20; sid:1000004; rev:1;)
alert udp any any -> any 51821 (msg:"WireGuard malformed packet"; dsize:<20; sid:1000005; rev:1;)
```

Включаем JSON-логи:

```
nano /etc/suricata/suricata.yaml
```

Находим строку и дописываем:

```
outputs:
  - eve-log:
      enabled: yes
      filetype: regular
      filename: /var/log/suricata/eve.json
      types:
        - alert
        - flow
```

Проверка конфига:

```
suricata -T -c /etc/suricata/suricata.yaml
```

Если ошибок нет, то все гуд, предупреждения тоже не страшно, главное чтоб написал, что успешный тестовый запуск.

Запускаем:

```
systemctl enable suricata
```

```
systemctl restart suricata
```

Чтобы проверить наличие логов, запускаем nmap и смотрим:

```
tail -f /var/log/suricata/eve.json
```

или

```
nano /var/log/suricata/eve.json
```

# LVM-массив на DC-STORAGE

Ставим пакеты:

```
apt install lvm2 cryptsetup inotify-tools -y
```

Смотрим названия дисков:

```
lsblk
```

Например, имеем ```/dev/vda```, ```/dev/vdb``` и ```/dev/vdc```. Работаем с ними.

Создаем физические тома:

```
pvcreate /dev/vda /dev/vdb /dev/vdc
```

Создаем группу томов:

```
vgcreate vg_crypto /dev/vda /dev/vdb /dev/vdc
```

Создаем логический том:

```
lvcreate -l 100%FREE -n lv_secure vg_crypto
```

Щифруем том:

```
cryptsetup luksFormat /dev/vg_crypto/lv_secure
```

Пароль: ```P@ssw0rd```

Открываем зашифрованный том:

```
cryptsetup luksOpen /dev/vg_crypto/lv_secure crypto_lv
```

Форматируем:

```
mkfs.ext4 /dev/mapper/crypto_lv
```

Добавляем ключевой файл:

```
dd if=/dev/random of=/root/luks_key bs=1024 count=4
```

```
chmod 400 /root/luks_key
```

Добавляем ключ в шифрованный раздел:

```
cryptsetup luksAddKey /dev/vg_crypto/lv_secure /root/luks_key
```

Настраиваем автоматическое монтирование:

```
nano /etc/crypttab
```

```
crypto_lv /dev/vg_crypto/lv_secure /root/luks_key luks
```

```
nano /etc/fstab
```

```
/dev/mapper/crypto_lv /crypto-folder ext4 defaults 0 2
```

```
systemctl daemon-reload
```

Создаем папку для монтирования:

```
mkdir /crypto-folder
```

Монтируем вручную (вдруг не перезапустят тачку и оно не примонтируется):

```
mount -a
```
