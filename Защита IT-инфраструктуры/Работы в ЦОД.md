# Настройка NFTABLES

> Обязательно смотрим открытые порты на каждом хосте. Если что - добавляем их в правила.
>
> Также в самом начале можем пока вставить в дефолтный конфиг только таблицу NAT ```table ip nat```.

На каждом роутере включаем IP-forward:

```
echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf
```

И перезапускаем:

```
sysctl -p
```

## DC-RTR-1 и DC-RTR-2

Включаем nftables:

```
systemctl enable nftables && systemctl restart nftables
```

Открываем конфиг:

```
nano /etc/nftables.conf
```

Вставляем правила:

```
table inet filter {
        chain input {
                type filter hook input priority filter; policy drop;

                # Разрешаем loopback
                iif lo accept

                # Разрешаем established/related
                ct state established,related accept

                # Разрешаем ICMP
                ip protocol icmp accept

                # Разрешаем VRRP
                ip protocol vrrp accept

                # Разрешаем SSH
                tcp dport 22 accept

                # Разрешаем WireGuard
                udp dport {51820, 51821} accept

                # Разрешаем OSPF (протокол 89)
                ip protocol ospf accept

                # Разрешаем NFS
                tcp dport 2049 accept
        }

        chain forward {
                # Форвардинг нужен для пересылки пакетов
                type filter hook forward priority filter; policy accept;
        }

        chain output {
                # Исходящие соединения разрешаем все
                type filter hook output priority filter; policy accept;
        }
}
table ip nat {
        chain prerouting {
                type nat hook prerouting priority filter; policy accept;
        }

        # PAT
        chain postrouting {
                type nat hook postrouting priority srcnat; policy accept;
                oifname "ens3" ip saddr 10.15.10.0/24 masquerade
        }
}
```

Перезапускаем nftables:

```
systemctl restart nftables
```

# Настройка VRRP между DC-RTR-1 и DC-RTR-2

Устанавливаем пакеты на оба роутера:

```
apt install keepalived -y
```

## DC-RTR-1

```
nano /etc/keepalived/keepalived.conf
```

```
vrrp_instance VI_0 {
	state MASTER
	interface ens4
	virtual_router_id 51
	priority 150
	advert_int 1
	virtual_ipaddress {
		10.15.10.1/24
	}
}
```

Перезапускаем сервис:

```
systelctm restart keepalived
```

## DC-RTR-2

```
nano /etc/keepalived/keepalived.conf
```

```
vrrp_instance VI_0 {
	state BACKUP
	interface ens4
	virtual_router_id 51
	priority 100
	advert_int 1
	virtual_ipaddress {
		10.15.10.1/24
	}
}
```

Перезапускаем сервис:

```
systelctm restart keepalived
```

# Настройка SSH

## Добавление пользователя cod_admin на всех хостах (DC-RTR-1, DC-RTR-2, DC-STORAGE, DC-MAILSERVER)

Добавляем пользователя cod_admin:

```
adduser cod_admin
```

Пароль:

```
P@ssw0rd
```

## sudo для cod_admin без пароля на DC-STORAGE и DC-MAILSERVER

Ставим пакет sudo:

```
apt install sudo -y
```

Открываем visudo:

```
visudo
```

Добавляем строку:

```
cod_admin ALL=(ALL:ALL) NOPASSWD: ALL
```

## Доступ SSH

### Генерация ключей:

На DC-STORAGE создаем ключи из-под cod_admin:

```
ssh-keygen
```

На все вопросы просто жмем ```Enter```.

Копируем ключи

- на DC-MAILSERVER:

```
ssh_copy_id cod_admin@10.15.10.100
```

- на DC-RTR-2:

```
ssh_copy_id cod_admin@10.15.10.2
```

- на DC-RTR-1:

```
ssh_copy_id cod_admin@10.15.10.3
```

Переносим ключи в ```/ssh_keys```:

```
sudo mv .ssh/* /ssh_keys/
```

```
rm -r .ssh
```

### Доступ по локальным интерфейсам:

**На DC-RTR-1:**

```
nano /etc/ssh/sshd_config
```

```
ListenAddress = 10.15.10.3
```

Перезапускаем сервис:

```
systemctl restart sshd
```

**На DC-RTR-2:**

```
nano /etc/ssh/sshd_config
```

```
ListenAddress = 10.15.10.2
```

Перезапускаем сервис:

```
systemctl restart sshd
```

**На DC-MAILSERVER:**

```
nano /etc/ssh/sshd_config
```

```
ListenAddress = 10.15.10.100
```

Перезапускаем сервис:

```
systemctl restart sshd
```

### Харденинг SSH на DC-RTR-1, DC-RTR-2, DC-MAILSERVER:

Создаем файл:

```
nano /etc/ssh/sshd_config.d/atom.conf
```

Прописываем следующие настройки:

```
PasswordAuthentication no
PubkeyAuthentication yes
AuthenticationMethods publickey
PermitRootLogin no
MaxStartups 3
ClientAliveInterval 300
ClientAliveCountMax 0
```

Перезапускаем сервис:

```
systemctl restart sshd
```

# Настройка Suricata

> Настройки оинаковые на DC-RTR-1 и на DC-RTR-2.

## Suricata

Ставим пакет:

```
apt install suricata -y
```

Включаем режим IDS и настраиваем интерфейсы:

```
nano /etc/suricata/suricata.yaml
```

```
af-packet:
  - interface: ens3
    cluster-id: 99
    cluster-type: cluster_flow
    defrag: yes
```

> Также внизу этого блока надо удалить строку ```- interface: default```.

Настраиваем домашнюю сеть:

```
nano /etc/suricata/suricata.yaml
```

Смотрим в начале строку ```HOME_NET``` и меняем на:

```
HOME_NET: "[10.15.10.0/24]"
```

Подключаем правила:

```
nano /etc/suricata/suricata.yaml
```

Находим строку ```rule-files:``` и добавляем ```- local.rules```, удаляем ```- suricata.rules```.

Создаем правила:

```
nano /etc/suricata/rules/local.rules
```

```
# 1. Сканирование портов
alert tcp any any -> $HOME_NET any (msg:"PORT SCAN SYN"; flags:S; threshold:type both, track by_src, count 10, seconds 5; sid:1000001; rev:1;)

# 2. Брутфорс SSH
alert tcp any any -> $HOME_NET 22 (msg:"SSH BRUTE FORCE"; flow:to_server; threshold:type both, track by_src, count 5, seconds 60; sid:1000002; rev:1;)

# LVM-массив на DC-STORAGE

Ставим пакеты:

```
apt install lvm2 cryptsetup inotify-tools -y
```

Смотрим названия дисков:

```
lsblk
```

Например, имеем ```/dev/vda```, ```/dev/vdb``` и ```/dev/vdc```. Работаем с ними.

Создаем физические тома:

```
pvcreate /dev/vda /dev/vdb /dev/vdc
```

Создаем группу томов:

```
vgcreate vg_crypto /dev/vda /dev/vdb /dev/vdc
```

Создаем логический том:

```
lvcreate -l 100%FREE -n lv_secure vg_crypto
```

Щифруем том:

```
cryptsetup luksFormat /dev/vg_crypto/lv_secure
```

Пароль: ```P@ssw0rd```

Открываем зашифрованный том:

```
cryptsetup luksOpen /dev/vg_crypto/lv_secure crypto_lv
```

Форматируем:

```
mkfs.ext4 /dev/mapper/crypto_lv
```

Добавляем ключевой файл:

```
dd if=/dev/random of=/root/luks_key bs=1024 count=4
```

```
chmod 400 /root/luks_key
```

Добавляем ключ в шифрованный раздел:

```
cryptsetup luksAddKey /dev/vg_crypto/lv_secure /root/luks_key
```

Настраиваем автоматическое монтирование:

```
nano /etc/crypttab
```

```
crypto_lv /dev/vg_crypto/lv_secure /root/luks_key luks
```

```
nano /etc/fstab
```

```
/dev/mapper/crypto_lv /crypto-folder ext4 defaults 0 2
```

```
systemctl daemon-reload
```

Создаем папку для монтирования:

```
mkdir /crypto-folder
```

Монтируем вручную (вдруг не перезапустят тачку и оно не примонтируется):

```
mount -a
```

# NFS-ресурсы

## На DC-STORAGE

Ставим пакет:

```
apt install nfs-kernel-server -y
```

Создаем каталоги для монтирования:

```
mkdir -p /storage/it
```

```
mkdir -p /storage/office
```

Настраиваем права:

```
chmod 1777 -R /storage/it
```

```
chmod 1777 -R /storage/office
```

```
chown root:ID_it -R /storage/it
```

```
chown root:ID_office -R /storage/office
```

Настраиваем каталоги для экспорта:

```
nano /etc/exports
```

```
/storage/it 192.168.1.0/24(rw,sync,no_subtree_check,root_squash)
/storage/office 192.168.1.0/24(rw,sync,no_subtree_check,root_squash)
```

Редактируем конфиг:

```
nano /etc/default/nfs-kernel-server
```

Добавляем в конец файла строки (это нужно для того, чтобы NFS использовал только 2049 порт):

```
RPCNFSDOPTS="--nfs-version 4"
NFS_DISABLE_V3=yes
```

Перезапускаем сервис:

```
exportfs -ra
```

```
systemctl restart nfs-server
```

## На MSK-WORKER и MSK-ADMINPC

Ставим пакеты:

```
apt install nfs-client libpam-mount -y

Редактируем конфиг:

```
nano /etc/security/pam_mount.conf.xml

В секции <pam_mount> пишем:

```
<volume fstype="nfs" server="10.15.10.150" path="/storage/it" mountpoint="~/Desktops/Desktop1/IT_Folder" options="vers=4" sgrp="it"/>
<volume fstype="nfs" server="10.15.10.150" path="/storage/office" mountpoint="~/Desktops/Desktop1/Office_Folder" options="vers=4" sgrp="office"/>
```

Готово, можем заходить под пользователями из нужных групп и проверять автомонтирование.

### Настройка LDAP

На ```MSK-DC1``` создаем нового доменного пользователя, который будет коннектиться к ldap для чтения пользователей:

```

### Proxy nginx

Настраиваем PAM для nginx:

```
nano /etc/pam.d/nginx
```

```
auth required pam_ldap.so
account required pam_ldap.so
```

Редачим основной конфиг:

```
nano /etc/nginx/nginx.conf
```

Раскомменчиваем или добавляем строку в блок ```http```:

```
server_names_hash_bucket_size 64;
```

Создаем каталог для SSL-сертификата:

```
mkdir /etc/nginx/ssl
```

Кладем в каталог ключ ```www.atomskills.ru.key``` и сертификат ```www.atomskills.ru.crt```.

Создаем конфиг сайта:

```
nano /etc/nginx/sites-available/homepage
```

```
server {
    listen 80;
    server_name www.effort.cool;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name www.effort.cool;

    ssl_certificate /etc/nginx/ssl/www.effort.cool.crt;
    ssl_certificate_key /etc/nginx/ssl/www.effort.cool.key;

    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    location / {
        proxy_pass http://127.0.0.1:3000;
        auth_pam "Restricted";
        auth_pam_service_name "nginx";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

Создаем ссылку на файл:

```
ln -s /etc/nginx/sites-available/homepage /etc/nginx/sites-enabled/
```

Удаляем дефолтный конфиг:

```
rm /etc/nginx/sites-enabled/default
```

Перезапускаем сервис:

```
systemctl restart nginx
```

Заходим на сайт ```https://www.atomskills.ru``` с любого хоста, чтоб он прогрузил все файлы. Затем возвращаемся на ```CLOUD-VM1``` и чистим файл:

```
nano /opt/homepage/config/bookmarks.yaml
```

Чтобы не было лишних ссылок.

# Работы на REMOTE-TERMINAL

## Запрет всех интерпретаторов, кроме bash

```
astra-interpreters-lock enable
```

Проверяем:

```
astra-interpreters-lock status
```

Должно быть ```АКТИВНО```.

> Блокировка интерпретаторов не распространяется на пользователей из группы ```astra-admin```. Будут запрещены только вызовы интерпретаторов из консоли, при этом в скриптах (#!/usr/bin/python3 и прочее) будет все работать.

## Отправка логов на YEKT-RTR

> Работы на YEKT-RTR (настройка rsync и веб-сайта) приведены в файле [Работы в Екатеринбурге](Работы%20в%20Екатеринбурге.md#сайт-с-логами-на-yekt-rtr).

Ставим пакет:

```
apt install rsync -y
```

Создаем скрипт для отправки логов:

```
nano /root/send_logs.sh
```

```
#!/bin/bash

TS=$(date +%H%M)
FILE="/tmp/auth.log-$TS"

cp /var/log/auth.log $FILE

rsync -avz $FILE loguser@88.8.8.27::logs --password-file=/root/.rsync_pass
```

Делаем файл исполняемым:

```
chmod +x /root/send_logs.sh
```

Записываем пароль в файл:

```
echo 'RsynC_PA$$' > /root/.rsync_pass
```

```
chmod 600 /root/.rsync_pass
```

Настраиваем cron:

```
crontab -e
```

```
0 * * * * /root/send_logs.sh
```

Пробуем ручной запуск:

```
/root/send_logs.sh
```

На ```YEKT-RTR``` проверяем наличие лога:

```
ls /var/www/logs
```

Либо смотрим на сайте.
