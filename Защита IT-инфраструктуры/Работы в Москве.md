# Настройка DHCP на MSK-RTR

Ставим пакет:

```
apt install isc-dhcp-server -y
```

Редактируем конфиги:

```
nano /etc/default/isc-dhcp-server
```

Проверяем строки в файле:

```
DHCPDv4_CONF=/etc/dhcp/dhcpd.conf # Основной файл конфигурации
INTERFACESv4="ens4" # Внутренний интерфейс
```

Редактируем основной конфиг:

```
nano /etc/dhcp/dhcpd.conf
```

```
subnet 192.168.1.0 netmask 255.255.255.0 {
    range 192.168.1.50 192.168.1.100;
    option routers 192.168.1.1;
    option domain-name-servers 192.168.1.2, 77.88.8.1;
    option domain-name "effort.cool";
    default-lease-time 600;
    max-lease-time 7200;
}
```

Перезапускаем сервис:

```
systemctl restart isc-dhcp-server
```

# Настройка Основного Центра сертификации на MSK-DC1

## Настройка ЦС

Проверяем, чтобы был ```openssl```, если нет, ставим:

```
apt install openssl -y
```

Создаем структуру каталога для ЦС:

```
mkdir -p /root/ca/{certs,crl,newcerts,private}
```

```
chmod 700 /root/ca/private/
```

```
touch /root/ca/index.txt
```

```
echo 1000 > /root/ca/serial
```

```
echo 1000 > /root/ca/crlnumber
```

Создаем конфиг ЦС:

```
nano /root/ca/openssl.cnf
```

```
[ ca ]
default_ca = CA_default

[ CA_default ]
dir               = /root/ca
certs             = $dir/certs
crl_dir           = $dir/crl
crlnumber         = $dir/crlnumber
database          = $dir/index.txt
new_certs_dir     = $dir/newcerts
serial            = $dir/serial

certificate       = $dir/certs/ca.crt
private_key       = $dir/private/ca.key

default_md        = sha256
default_days      = 825
default_crl_days  = 30
copy_extensions   = copy
unique_subject    = no
policy            = policy_loose
x509_extensions   = v3_ca

[ policy_loose ]
countryName             = optional
stateOrProvinceName     = optional
localityName            = optional
organizationName        = optional
commonName              = supplied

[ req ]
default_bits        = 4096
default_md          = sha256
distinguished_name  = req_distinguished_name
x509_extensions     = v3_ca

[ req_distinguished_name ]
countryName         = Country Name
stateOrProvinceName = State
localityName        = City
organizationName    = Organization
commonName          = Common Name

[ v3_ca ]
subjectKeyIdentifier   = hash
authorityKeyIdentifier = keyid:always,issuer
basicConstraints       = critical, CA:true
keyUsage               = critical, digitalSignature, cRLSign, keyCertSign

[ v3_intermediate_ca ]
subjectKeyIdentifier   = hash
authorityKeyIdentifier = keyid:always,issuer
basicConstraints       = critical, CA:true, pathlen:0
keyUsage               = critical, digitalSignature, cRLSign, keyCertSign
```

Генерируем закрытый ключ ЦС:

```
openssl genrsa -out /root/ca/private/ca.key 4096
```

Создаем сертификат ЦС:

```
openssl req -x509 -config /root/ca/openssl.cnf -new -nodes -key /root/ca/private/ca.key -sha256 -days 3650 -out /root/ca/certs/ca.crt -subj "/C=RU/ST=Moskovskaya/L=Moskva/O=Cool CA/CN=Cool CA"
```

## Добавление сертификата ЦС в доверенные:

Копируем ```/root/ca/certs/ca.crt``` на все хосты.

Добавляем ```ca.crt``` в доверенные:

```
cp ca.crt /usr/local/share/ca-certificates/ca.crt
```

```
update-ca-certificates
```

Проверяем, что доверяем сертификату:

```
openssl verify /usr/local/share/ca-certificates/ca.crt
```

*Должно быть ```OK```.*

> Чтобы в брузере не выскакивала ошибка сертификата, нужно добавить ```ca.crt``` в доверенные конкретно в браузере!

## Выпуск сертификатов для web-сервисов

### www.effort.cool (Homepage на CLOUD-VM1)

Генерим ключ:

```
openssl genrsa -out effort.cool.key 4096
```

Создаем конфиг:

```
nano effort.cool.cnf
```

```
[ req ]
distinguished_name = req_distinguished_name
req_extensions = server_cert

[ req_distinguished_name ]

[ server_cert ]
basicConstraints = CA:FALSE
keyUsage = digitalSignature, keyEncipherment
extendedKeyUsage = serverAuth
subjectAltName = @alt_names

[ alt_names ]
DNS.1 = www.effort.cool
DNS.2 = effort.cool
```

Делаем запрос сертификата:

```
openssl req -new -key effort.cool.key -out effort.cool.csr -config effort.cool.cnf -subj "/C=RU/ST=Moskovskaya/L=Moskva/O=Cool CA/CN=www.effort.cool"
```

Подписываем сертификат:

```
openssl ca -config /root/ca/openssl.cnf -extfile effort.cool.cnf -extensions server_cert -notext -in effort.cool.csr -out effort.cool.crt
```

Затем переносим ```www.effort.cool.key``` и ```www.effort.cool.crt``` на CLOUD-VM1 и используем для настройки HTTPS.
