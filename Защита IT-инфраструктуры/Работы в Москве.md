# Настройка DHCP на MSK-RTR

Ставим пакет:

```
apt install isc-dhcp-server -y
```

Редактируем конфиги:

```
nano /etc/default/isc-dhcp-server
```

Проверяем строки в файле:

```
DHCPDv4_CONF=/etc/dhcp/dhcpd.conf # Основной файл конфигурации
INTERFACESv4="ens4" # Внутренний интерфейс
```

Редактируем основной конфиг:

```
nano /etc/dhcp/dhcpd.conf
```

```
subnet 192.168.1.0 netmask 255.255.255.0 {
    range 192.168.1.50 192.168.1.100;
    option routers 192.168.1.1;
    option domain-name-servers 192.168.1.2, 77.88.8.1;
    option domain-name "effort.cool";
    default-lease-time 600;
    max-lease-time 7200;
}
```

Перезапускаем сервис:

```
systemctl restart isc-dhcp-server
```

# Настройка Основного Центра сертификации на MSK-DC1

## Настройка ЦС

Проверяем, чтобы был ```openssl```, если нет, ставим:

```
apt install openssl -y
```

Создаем структуру каталога для ЦС:

```
mkdir -p /root/ca/{certs,crl,newcerts,private}
```

```
chmod 700 /root/ca/private/
```

```
touch /root/ca/index.txt
```

```
echo 1000 > /root/ca/serial
```

```
echo 1000 > /root/ca/crlnumber
```

Создаем конфиг ЦС:

```
nano /root/ca/openssl.cnf
```

```
[ ca ]
default_ca = CA_default

[ CA_default ]
dir               = /root/ca
certs             = $dir/certs
crl_dir           = $dir/crl
crlnumber         = $dir/crlnumber
database          = $dir/index.txt
new_certs_dir     = $dir/newcerts
serial            = $dir/serial

certificate       = $dir/certs/ca.crt
private_key       = $dir/private/ca.key

default_md        = sha256
default_days      = 825
default_crl_days  = 30
copy_extensions   = copy
unique_subject    = no
policy            = policy_loose
x509_extensions   = v3_ca

[ policy_loose ]
countryName             = optional
stateOrProvinceName     = optional
localityName            = optional
organizationName        = optional
commonName              = supplied

[ req ]
default_bits        = 4096
default_md          = sha256
distinguished_name  = req_distinguished_name
x509_extensions     = v3_ca

[ req_distinguished_name ]
countryName         = Country Name
stateOrProvinceName = State
localityName        = City
organizationName    = Organization
commonName          = Common Name

[ v3_ca ]
subjectKeyIdentifier   = hash
authorityKeyIdentifier = keyid:always,issuer
basicConstraints       = critical, CA:true
keyUsage               = critical, digitalSignature, cRLSign, keyCertSign

[ v3_intermediate_ca ]
subjectKeyIdentifier   = hash
authorityKeyIdentifier = keyid:always,issuer
basicConstraints       = critical, CA:true, pathlen:0
keyUsage               = critical, digitalSignature, cRLSign, keyCertSign
```

Генерируем закрытый ключ ЦС:

```
openssl genrsa -out /root/ca/private/ca.key 4096
```

Создаем сертификат ЦС:

```
openssl req -x509 -config /root/ca/openssl.cnf -new -nodes -key /root/ca/private/ca.key -sha256 -days 3650 -out /root/ca/certs/ca.crt -subj "/C=RU/ST=Moskovskaya/L=Moskva/O=Cool CA/CN=Cool CA"
```

## Добавление сертификата ЦС в доверенные:

Копируем ```/root/ca/certs/ca.crt``` на все хосты.

Добавляем ```ca.crt``` в доверенные:

```
cp ca.crt /usr/local/share/ca-certificates/ca.crt
```

```
update-ca-certificates
```

Проверяем, что доверяем сертификату:

```
openssl verify /usr/local/share/ca-certificates/ca.crt
```

*Должно быть ```OK```.*

> Чтобы в брузере не выскакивала ошибка сертификата, нужно добавить ```ca.crt``` в доверенные конкретно в браузере!

## Выпуск сертификатов для web-сервисов

### msk-dc1.effort.cool (Free IPA HTTPS)

Генерим ключ:

```
openssl genrsa -out msk-dc1.effort.cool.key 4096
```

Создаем конфиг:

```
nano msk-dc1.effort.cool.cnf
```

```
[ req ]
distinguished_name = req_distinguished_name
req_extensions = server_cert

[ req_distinguished_name ]

[ server_cert ]
basicConstraints = CA:FALSE
keyUsage = digitalSignature, keyEncipherment
extendedKeyUsage = serverAuth
subjectAltName = @alt_names

[ alt_names ]
DNS.1 = msk-dc1.effort.cool
IP.1 = 192.168.1.2
```

Делаем запрос сертификата:

```
openssl req -new -key msk-dc1.effort.cool.key -out msk-dc1.effort.cool.csr -config msk-dc1.effort.cool.cnf -subj "/C=RU/ST=Moskovskaya/L=Moskva/O=Cool CA/CN=msk-dc1.effort.cool"
```

Подписываем сертификат:

```
openssl ca -config /root/ca/openssl.cnf -extfile msk-dc1.effort.cool.cnf -extensions server_cert -notext -in msk-dc1.effort.cool.csr -out msk-dc1.effort.cool.crt
```

Собираем PKCS#12:

```
openssl pkcs12 -export -out ipa.p12 -inkey msk-dc1.effort.cool.key -in msk-dc1.effort.cool.crt -certfile /root/ca/certs/ca.crt
```

Устанавливаем корневой CA в IPA:

```
ipa-cacert-manage install /root/ca/certs/ca.crt
```

Обновляем сервисы:

```
ipa-certupdate
```

Устанавливаем сертификат:

```
ipa-server-certinstall -w -d ipa.p12
```

Перезапускаем IPA:

```
ipactl restart
```

# Работа в домене Free IPA

## Импорт пользователей из файла

Вытаскиваем названия групп из файла:

```
cut -d',' -f3 /root/users.csv | tail -n +2 | sort | uniq
```

Добавляем все группы вручную в Free IPA:

```
Идентификация -> Группы -> Добавить -> Вводим имя группы -> "Добавить и добавить еще"
```

Далее создаем скрипт для импорта пользователей:

```
nano import.sh
```

```
#!/bin/bash
while IFS="," read -r LastName FirstName Groups
do
	LOGIN=$LastName
	echo P@ssw0rd | echo P@ssw0rd | ipa user-add "${LOGIN}" --first="$FirstName" --last="${LastName}" --password
	ipa user-mod "$LOGIN" --setattr=KrbPasswordExpiration=20261111111111Z
	ipa group-add-member $Groups --users=$LOGIN
done < <(tail -n +2 /root/users.csv)
```

Даем права на выполнение:

```
chmod +x import.sh
```

Получаем Kerberos билет админа домена:

```
kinit admin
```

Импортируем пользователей:

```
./import.sh
```

## Резервное копирование LDAP

Создаем файл с паролем:

```
mkdir -p /root/.backup
```

```
chmod 700 /root/.backup
```

```
echo 'BackupPa$$' > /root/.backup/backup.key
```

```
chmod 600 /root/.backup/backup.key
```

Пишем скрипт для резервного копирования:

```
nano /usr/local/bin/ipa-ldap-backup.sh
```

```
#!/bin/bash
set -euo pipefail

DATE=$(date +%F-%H-%M)
WORKDIR=$(mktemp -d /tmp/ipa_backup.XXXXXX)

ARCHIVE="/mnt/backup-user.backup"
DECRYPTED="$WORKDIR/archive.tar"
KEY_FILE="/root/.backup/backup.key"
LDIF_FILE="$WORKDIR/ldap_$DATE.ldif"

LOG="/var/log/ipa-backup.log"
exec >> "$LOG" 2>&1

echo "[*] $(date) Start backup"

# Проверка ключа
[ -f "$KEY_FILE" ] || { echo "[!] Key missing"; exit 1; }

# 1. Экспорт LDAP (онлайн)
echo "[*] Export LDAP via ldapsearch"

ldapsearch -LLL -Y GSSAPI -b "" -s sub "(objectClass=*)" > "$LDIF_FILE"

[ -s "$LDIF_FILE" ] || { echo "[!] LDIF empty"; exit 1; }

# 2. Если архив существует — расшифровываем
if [ -f "$ARCHIVE" ]; then
    echo "[*] Decrypt existing archive"

    openssl enc -d -aes-256-cbc -pbkdf2 -in "$ARCHIVE" -out "$DECRYPTED" -pass file:"$KEY_FILE"

    mkdir -p "$WORKDIR/extracted"
    tar -xf "$DECRYPTED" -C "$WORKDIR/extracted"
else
    echo "[*] No existing archive, creating new"
    mkdir -p "$WORKDIR/extracted"
fi

# 3. Добавляем новый LDIF
cp "$LDIF_FILE" "$WORKDIR/extracted/"

# 4. Пересобираем архив
echo "[*] Repack archive"
tar -cf "$DECRYPTED" -C "$WORKDIR/extracted" .

# 5. Шифруем обратно
echo "[*] Encrypt archive"
openssl enc -aes-256-cbc -salt -pbkdf2 -in "$DECRYPTED" -out "$ARCHIVE" -pass file:"$KEY_FILE"

# 6. Очистка
rm -rf "$WORKDIR"

echo "[+] Done"
```

Выдаем права на выполнение:

```
chmod 700 /usr/local/bin/ipa-ldap-backup.sh
```

Создаем сервис:

```
nano /etc/systemd/system/ipa-ldap-backup.service
```

```
[Unit]
Description=FreeIPA LDAP Backup

[Service]
Type=oneshot
ExecStartPre=/usr/bin/kinit -k
ExecStart=/usr/local/bin/ipa-ldap-backup.sh
```

Создаем таймер:

```
nano /etc/systemd/system/ipa-ldap-backup.timer
```

```
[Unit]
Description=Daily FreeIPA LDAP Backup

[Timer]
OnCalendar=*-*-* 02:00:00
Persistent=true

[Install]
WantedBy=timers.target
```

Применяем изменения и включаем сервис:

```
systemctl daemon-reexec
```

```
systemctl daemon-reload
```

```
systemctl enable --now ipa-ldap-backup.timer
```

Ручной запуск:

```
systemctl start ipa-ldap-backup
```
